# MPD 多平台瘦身构建说明

## 一、概述

在 chroot 环境中编译 MPD + mpc，并编译瘦身版 ffmpeg（仅音频解码器，无视频/GPU 依赖），产物打包为独立可运行的 tgz 压缩包。

### 目标平台

| 平台 | libc | 构建方式 | 产物大小参考 |
|------|------|---------|------------|
| x86_64-glibc | glibc | 宿主编译（无 chroot） | ~16M |
| x86_64-musl | musl | Alpine x86_64 chroot | ~14M |
| arm64-glibc | glibc | Debian arm64 chroot | ~14M |
| arm64-musl | musl | Alpine arm64 chroot | ~11M |
| armv7-glibc | glibc | Debian armv7 chroot | ~12M |

### 产物结构

```
mpd-player-linux-{ARCH}-{LIBC}.tgz
├── mpd           # 包装脚本（设置 LD_LIBRARY_PATH 后 exec mpd.real）
├── mpc           # 包装脚本（同上）
├── mpd.real      # MPD ELF 可执行文件
├── mpc.real      # mpc ELF 可执行文件
└── lib/          # 所有依赖库
```

### 瘦身原理

**问题**：Alpine 的 `ffmpeg-dev` 包（Debian 的 `libavcodec-dev` 等）包含完整的 ffmpeg：视频编解码器（x265 19M、aom 7.3M、x264 2.2M）、GPU 渲染管线（SPIR-V、vulkan、placehbo ~6.8M）等，MPD 完全用不到。

**方案**：从源码编译 ffmpeg，用 `--disable-everything` 只保留音频解码器和 HLS 流协议。将编译产物安装到 `/usr/local`，MPD 的 meson 通过 `PKG_CONFIG_PATH` 找到。

**核心约束（musl 平台）**：musl 不支持 RTLD_LAZY（延迟绑定），加载时强制解析全部符号。因此 **musl 平台不能用 patchelf 删 DT_NEEDED 来瘦身**，必须全量打包 ldd 输出的所有库。瘦身只能通过减少 ffmpeg 编译进来的库来实现。

## 二、宿主环境准备

```bash
apt-get update

# 基础构建工具
apt-get install -y meson ninja-build curl build-essential pkg-config

# QEMU 用户态模拟（用于运行 arm64/armv7 二进制）
```

## 三、Chroot 环境搭建

### 3.1 x86_64-musl（Alpine x86_64 chroot）

```bash
mkdir -p /workspace/chroots/x86_64-musl
curl -LO https://dl-cdn.alpinelinux.org/alpine/v3.20/releases/x86_64/alpine-minirootfs-3.20.3-x86_64.tar.gz
tar -xzf alpine-minirootfs-3.20.3-x86_64.tar.gz -C /workspace/chroots/x86_64-musl
```

**chroot 内初始化**：

```bash
chroot /workspace/chroots/x86_64-musl /bin/sh -c "
echo 'nameserver 8.8.8.8' > /etc/resolv.conf
apk update
apk add alpine-base bash build-base gcc g++ meson ninja curl git linux-headers
apk add alsa-lib-dev pulseaudio-dev pipewire-dev expat-dev curl-dev
apk add flac-dev libmad-dev mpg123-dev libvorbis-dev opus-dev
apk add libsndfile-dev sqlite-dev libid3tag-dev
"
```

**挂载 /dev 和 /proc**（Alpine chroot 内 git clone 和某些工具需要）：

```bash
for d in dev proc; do mount --bind /$d /workspace/chroots/x86_64-musl/$d; done
```

验证：

```bash
chroot /workspace/chroots/x86_64-musl /bin/bash -c "uname -m && gcc --version | head -1"
# 输出: x86_64
```

## 四、构建脚本说明

### 4.1 脚本列表

| 脚本 | 运行位置 | 依赖 | 目标 |
|------|---------|------|------|
| `build-linux-x86_64-glibc.sh` | **宿主编译** | 宿主机 | x86_64 glibc |
| `build-linux-x86_64-musl.sh` | 宿主编译，chroot 内构建 | Alpine x86_64 chroot | x86_64 musl |
| `build-linux-arm64-glibc.sh` | 宿主编译，chroot 内构建 | Debian arm64 chroot | ARM64 glibc |
| `build-linux-arm64-musl.sh` | 宿主编译，chroot 内构建 | Alpine arm64 chroot | ARM64 musl |
| `build-linux-armv7-glibc.sh` | 宿主编译，chroot 内构建 | Debian armv7 chroot | ARMv7 glibc |

### 4.2 脚本通用流程

每个脚本的构建流程相同：

```
下载 MPD/mpc/ffmpeg 源码
    │
    ▼
安装构建依赖（apk/apt 安装 soxr-dev/nasm 等）
    │
    ▼
编译瘦身版 ffmpeg 到 /usr/local（仅首次，缓存后跳过）
    └─ configure --disable-everything --enable-decoder=音频... --enable-demuxer=... --enable-protocol='file,http,https,hls...'
    │
    ▼
meson setup → ninja build MPD
    └─ -Dffmpeg=enabled -Dsoxr=enabled  (meson 通过 pkg-config 发现 /usr/local/lib 下的 slim ffmpeg)
    │
    ▼
meson setup → ninja build mpc
    │
    ▼
ldd 分析依赖 → 复制所有 .so 到 lib/
    └─ grep -o '/[^ ]*\.so[^ ]*'  (捕获 /usr/lib、/lib、/usr/local/lib 所有路径)
    │
    ▼
创建包装脚本（设定 LD_LIBRARY_PATH）+ tar 打包
```

### 4.3 关键 meson 选项

```
-Dffmpeg=enabled    # 通过 ffmpeg 支持 HLS/m3u8、AAC、WMA、APE、AC3 等扩展格式
-Dsoxr=enabled      # 高质量重采样器（soxr）
-Ddbus=disabled     # 禁用 D-Bus（减小体积）
-Dsystemd=disabled  # 禁用 systemd（减小体积）
-Dipv6=disabled     # 禁用 IPv6
```

### 4.4 slim ffmpeg 配置详解

```bash
./configure --prefix=/usr/local \
    --disable-everything \
    --enable-decoder='mp3*,aac*,vorbis,opus,flac,alac,wmav*,wmapro*,wmalossless,wmavoice' \
    --enable-decoder='ac3,eac3,dts,truehd,mlp' \
    --enable-decoder='ape,wavpack*,tta,pcm*,adpcm*' \
    --enable-decoder='cook,amrnb,amrwb,gsm*,qcelp,atrac*' \
    --enable-demuxer='mp3,aac,ogg,flac,wav,aiff,matroska,asf,ac3,eac3,dts,hls' \
    --enable-demuxer='ape,wv,tta,mp4,mov,au,pcm*,w64' \
    --enable-parser='mpegaudio,aac,vorbis,opus,flac,ac3' \
    --enable-protocol='file,pipe,data,http,https,tls,crypto' \
    --enable-filter=aresample \
    --enable-shared --disable-static --disable-doc --disable-programs
```

**重点**：
- `--disable-everything` 起点，然后按需开启
- `--disable-programs` 不编译 ffmpeg/ffplay/ffprobe 命令行工具
- `--enable-protocol='http,https,tls,crypto'` 支持 HLS 电台流
- `--enable-demuxer=hls` HLS m3u8 流解析
- `--enable-filter=aresample` 音频重采样（MPD crossfade 需要）
- 安装到 `/usr/local`，与系统包管理器路径隔离

## 七、常见错误与解决方案

### 7.1 chroot 内 apt-get 无法联网

chroot 缺少 DNS 配置：

```bash
# 从宿主编译运行以下命令
cp /etc/resolv.conf /workspace/chroots/arm64-glibc/etc/resolv.conf
```

### 7.2 chroot 内 apk 报网络错误

Alpine minirootfs 默认没有 /etc/resolv.conf，需要手动写入：

```bash
echo 'nameserver 8.8.8.8' > /workspace/chroots/x86_64-musl/etc/resolv.conf
```

### 7.3 QEMU binfmt 未正确注册

执行 arm64 二进制时报 `Exec format error`：

```bash
update-binfmts --enable qemu-aarch64
update-binfmts --enable qemu-arm
systemctl restart systemd-binfmt
# 或重新安装
apt-get install --reinstall qemu-user-static binfmt-support
```

### 7.4 Alpine chroot 内 git clone 失败（/dev/urandom 不可用）

```bash
mount --bind /dev /workspace/chroots/x86_64-musl/dev
mount --bind /proc /workspace/chroots/x86_64-musl/proc
```

### 7.5 musl 平台 ldd 输出只匹配 /usr/lib/

musl 的 ldd 输出中部分库（如 libcrypto、libssl、libz）路径为 `/lib/`，有的在 `/usr/local/lib/`。grep 必须使用 `'/[^ ]*\.so[^ ]*'` 匹配所有前缀：

```bash
ldd mpd.real | grep -o '/[^ ]*\.so[^ ]*'
```

### 7.6 musl 平台不能用 patchelf 瘦身

musl 不支持 RTLD_LAZY，加载时强制解析所有符号。删除 DT_NEEDED 后 musl ld 遍历 relocation 发现缺失符号立即报错退出。**musl 平台必须全量打包 ldd 输出的所有库，不能使用 patchelf + 白名单过滤**。

### 7.7 首次构建慢

arm64/armv7 平台在 QEMU 模拟下编译极慢（约 1 分钟 / 文件）。slim ffmpeg 编译约 5-10 分钟（仅有音频解码器），MPD 编译约 20-30 分钟。后续构建因为 ffmpeg 已安装到 `/usr/local` 会跳过，只编译 MPD。

### 7.8 MPD 构建时找不到 ffmpeg（libavcodec 等）

meson 通过 pkg-config 查找 ffmpeg。slim ffmpeg 的 .pc 文件在 `/usr/local/lib/pkgconfig/`。构建时需设置：

```bash
export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig:$PKG_CONFIG_PATH
```

脚本中已包含此设置，如果在 chroot 中手动调试，需要确保 PKG_CONFIG_PATH 正确。

### 7.9 libsoxr-dev 包名不同

- Debian/Ubuntu (apt)：`libsoxr-dev`
- Alpine (apk)：`soxr-dev`

### 7.10 mpc 版本

mpc 最新版本为 0.35（不是 0.45，0.45 不存在于 GitHub releases）。

### 7.11 glibc平台构建完成后ldd收集grep写法

glibc 平台的 ldd 输出中，库依赖路径可能是 `/lib/x86_64-linux-gnu/` 或 `/usr/lib/x86_64-linux-gnu/`。使用 `grep -o '/[^ ]*\.so[^ ]*'` 即可捕获所有路径，保留目录结构（lib/x86_64-linux-gnu/）。

### 7.12 ICU 库被静默静态链接（体积膨胀 ~3MB）

**现象**：瘦身版产物中意外出现 `libicudata.so`、`libicui18n.so`、`libicuuc.so` 三个 ICU 库。

**原因**：MPD meson 构建默认自动检测 ICU，即使未安装 `libicu-dev`，`-Dicu=auto` 也会在系统中寻找。如果宿主系统装过 `libicu-dev`（Debian `build-essential` 可能间接引入），meson 会静默启用 ICU 并以静态方式链接。ICU 仅用于 Unicode 排序和字符集转换，**播放功能完全不需要**，却引入约 3MB 的体积膨胀。

**修复**：meson setup 命令必须显式添加 `-Dicu=disabled`：

```bash
# 正确写法
meson setup build \
  -Dffmpeg=enabled \
  -Dicu=disabled \
  ...
```

**为什么不能只卸载 `libicu-dev`**：因为 meson 的 `auto` 检测逻辑不确定——即使当前 chroot 没有安装 ICU，某些平台的交叉编译工具链或容器基础镜像可能已经预装了 ICU。显式 `disabled` 是唯一可靠的方式。

**验证**：构建后在产物 `lib/` 目录中搜索：

```bash
ls lib/ | grep -i icu
# 期望输出: 空
```

如果输出非空，说明 ICU 未被禁用，检查 meson 的 configure 日志确认编译选项生效。

## 八、验证方法

```bash
mkdir -p /tmp/verify

```

### 7.13 glibc chroot 中瘦身 ffmpeg 被系统 ffmpeg dev 包覆盖

**现象**：glibc 平台构建产物异常大（60-65M），tgz 中包含 `libdav1d`、`libaom`、`libx264`、`librav1e`、`libvpx`、`libjxl` 等视频库。

**原因**：Debian glibc chroot 在搭建时 apt 安装了 `libavcodec-dev`、`libavformat-dev`、`libavutil-dev` 等系统 ffmpeg dev 包（版本 5.x）。这些包的 pkgconfig 文件位于 `/usr/lib/<arch>/pkgconfig/`，瘦身 ffmpeg 的 pkgconfig 在 `/usr/local/lib/pkgconfig/`。

脚本使用 `PKG_CONFIG_PATH=/usr/local/lib/pkgconfig:$PKG_CONFIG_PATH` 理论上应该优先找到瘦身版。但 meson 的包查找机制可能因缓存或搜索顺序问题仍会命中系统版本（meson 日志显示 `libavcodec found: YES 59.37.100 (cached)` 而非 60.31.102）。

**验证**：在 chroot 中检查 pkgconfig 文件：

```bash
# 检查是否有系统 ffmpeg .pc 文件
chroot /workspace/chroots/arm64-glibc sh -c 'ls /usr/lib/aarch64-linux-gnu/pkgconfig/libav*.pc'
# 正确状态：无输出（已移除）
# 错误状态：列出 libavcodec.pc / libavformat.pc / libavutil.pc
```

**修复**：从 glibc chroot 中移除系统 ffmpeg dev 包，只保留瘦身版：

```bash
chroot /workspace/chroots/arm64-glibc apt-get remove -y libavcodec-dev libavformat-dev libavutil-dev
chroot /workspace/chroots/armv7-glibc apt-get remove -y libavcodec-dev libavformat-dev libavutil-dev
```

移除后，chroot 中只剩 `/usr/local/lib/pkgconfig/` 下的瘦身 ffmpeg pkgconfig 文件，meson 只能找到瘦身版。

**注意**：不要用 `PKG_CONFIG_LIBDIR=/usr/local/lib/pkgconfig` 替代 `PKG_CONFIG_PATH`，因为 `LIBDIR` 会**完全替换**所有 pkgconfig 搜索路径，导致 meson 找不到 alsa/pulse/expat 等系统的 .pc 文件，构建失败。
