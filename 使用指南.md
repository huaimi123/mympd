# MPD 二进制构建 - 使用指南

## 📁 文件说明

### 构建脚本（5个）
- `build-linux-arm64-glibc.sh` - ARM64 glibc 平台构建
- `build-linux-arm64-musl.sh` - ARM64 musl 平台构建  
- `build-linux-armv7-glibc.sh` - ARMv7 glibc 平台构建
- `build-linux-x86_64-glibc.sh` - x86_64 glibc 平台构建
- `build-linux-x86_64-musl.sh` - x86_64 musl 平台构建

### 辅助脚本（1个）
- `build-all.sh` - 一键构建所有平台

### GitHub Actions（1个）
- `.github/workflows/build.yml` - GitHub Actions 自动构建工作流

### 文档（1个）
- `README.md` - 完整的使用说明

## 🚀 快速开始

### 方式一：GitHub Actions 自动构建（推荐）

1. **创建 GitHub 仓库**:
   - 访问 https://github.com/new
   - 创建新仓库，选择 "Public" 或 "Private"
   - 不要初始化 README、.gitignore 或 license

2. **上传文件到 GitHub**:
   ```bash
   # 在当前目录执行
   cd /workspace
   
   # 初始化 Git 仓库
   git init
   git config user.name "Your Name"
   git config user.email "your.email@example.com"
   
   # 添加所有文件
   git add mpd二进制构建/
   
   # 提交
   git commit -m "Add MPD build scripts for all platforms"
   
   # 添加远程仓库（替换为你的仓库地址）
   git remote add origin https://github.com/your-username/your-repo.git
   
   # 推送到 GitHub
   git branch -M main
   git push -u origin main
   ```

3. **启用 GitHub Actions**:
   - 访问你的 GitHub 仓库
   - 点击 "Actions" 标签
   - 如果看到黄色横幅，点击 "I understand my workflows, go ahead and enable them"

4. **手动触发构建**:
   - 访问 "Actions" 标签
   - 选择 "MPD Binary Build" 工作流
   - 点击 "Run workflow" 按钮
   - 选择分支，点击 "Run workflow"

5. **下载构建产物**:
   - 等待构建完成（约 1-2 小时）
   - 访问 Actions 页面中完成的构建
   - 在 "Artifacts" 部分下载对应的平台包

6. **获取 Release 版本**（可选）:
   - 推送到 `main` 分支会自动创建 Release
   - 访问 "Releases" 标签下载最新版本

### 方式二：本地手动构建

#### 1. 安装必要工具

```bash
sudo apt-get update
sudo apt-get install -y \
  debootstrap qemu-user-static \
  curl git meson ninja-build \
  patchelf pkg-config
```

#### 2. 准备 Chroot 环境（仅 ARM 平台需要）

```bash
# ARM64 glibc
sudo mkdir -p /workspace/chroots/arm64-glibc
sudo debootstrap --arch=arm64 bookworm /workspace/chroots/arm64-glibc http://deb.debian.org/debian
sudo cp /usr/bin/qemu-aarch64-static /workspace/chroots/arm64-glibc/usr/bin/

# ARM64 musl
sudo mkdir -p /workspace/chroots/arm64-musl
curl -L -o /tmp/alpine-rootfs.tar.gz "https://dl-cdn.alpinelinux.org/alpine/latest-stable/releases/aarch64/alpine-minirootfs-3.23.0-aarch64.tar.gz"
sudo tar -xzf /tmp/alpine-rootfs.tar.gz -C /workspace/chroots/arm64-musl
sudo cp /usr/bin/qemu-aarch64-static /workspace/chroots/arm64-musl/usr/bin/

# ARMv7 glibc
sudo mkdir -p /workspace/chroots/armv7-glibc
sudo debootstrap --arch=armhf bookworm /workspace/chroots/armv7-glibc http://deb.debian.org/debian
sudo cp /usr/bin/qemu-arm-static /workspace/chroots/armv7-glibc/usr/bin/
```

#### 3. 构建单个平台

```bash
cd "mpd二进制构建"

# x86_64 glibc（无需 chroot）
./build-linux-x86_64-glibc.sh

# ARM 平台（需要 sudo）
sudo ./build-linux-arm64-glibc.sh
sudo ./build-linux-arm64-musl.sh
sudo ./build-linux-armv7-glibc.sh
```

#### 4. 构建所有平台

```bash
cd "mpd二进制构建"
./build-all.sh
```

#### 5. 查找构建产物

```bash
# 查看所有构建产物
find workspace/ -name "*.tgz" -exec ls -lh {} \;

# 或者查看特定平台
ls -lh workspace/linux-arm64-glibc/archive/
```

## 📦 使用构建产物

1. **下载对应的平台压缩包**
2. **解压缩**:
   ```bash
   tar -xzf mpd-player-linux-arm64-glibc.tgz
   ```

3. **运行 MPD**:
   ```bash
   # 启动 MPD 服务
   ./mpd
   
   # 运行 MPC 客户端
   ./mpc
   ```

## ⚠️ 重要注意事项

1. **文件权限**: 确保脚本有执行权限
   ```bash
   chmod +x mpd二进制构建/*.sh
   ```

2. **Chroot 环境**: ARM 平台需要预先生成 chroot 环境

3. **构建时间**: 
   - x86_64 平台：~10-30 分钟
   - ARM 平台：~60-90 分钟（QEMU 模拟）

4. **磁盘空间**: 确保有足够的磁盘空间（至少 10GB）

5. **网络连接**: 需要稳定的网络连接下载源代码

## 🐛 故障排除

### Chroot 相关问题

```bash
# 检查 QEMU 是否正常工作
qemu-aarch64-static --version

# 检查 chroot 目录
ls -la /workspace/chroots/

# 手动挂载文件系统
sudo mount --bind /dev /workspace/chroots/arm64-glibc/dev
sudo mount --bind /proc /workspace/chroots/arm64-glibc/proc
```

### 依赖问题

```bash
# 检查构建依赖
which meson ninja
meson --version
ninja --version
```

### 权限问题

```bash
# 检查脚本权限
ls -la mpd二进制构建/*.sh

# 重新添加执行权限
chmod +x mpd二进制构建/*.sh
```
